## 1. 平台分层与角色职责

### 1.1 三层架构

```
系统（System）          ← 全局配置：用户、组织、资源池、Agent Token
  └── 组织（Organization）  ← 业务隔离：成员、项目、模板、组织架构
        └── 项目（Project）  ← 测试工作单元：用例、计划、缺陷、环境
```

**治理原则**：

- **系统层**管「能不能建组织、谁能登录、资源池够不够」
- **组织层**管「这个事业部有哪些人、有哪些项目、模板怎么定」
- **项目层**管「这个项目谁测、环境在哪、成员权限怎么分」

### 1.2 三类管理人员职责矩阵

| 职责 | 系统管理员 | 组织管理员 | 项目管理员 |
|------|:--:|:--:|:--:|
| 创建 / 禁用 / 删除组织 | ✅ | ❌ | ❌ |
| 切换组织上下文（全局） | ✅ | ✅（本组织） | ✅（有权限的组织） |
| 系统用户 CRUD | ✅ | ❌ | ❌ |
| 资源池管理 | ✅ | ❌ | ❌ |
| Agent Token 管理 | ✅ | ❌ | ❌ |
| 企微通讯录同步配置 | ✅ | 本组织只读查看 | ❌ |
| 组织成员添加 / 移除 | ✅ | ✅ | ❌ |
| 组织用户组 / 模板 | ✅ | ✅ | ❌ |
| 创建 / 归档组织内项目 | ✅ | ✅ | ❌ |
| 项目成员与角色分配 | ✅ | ✅ | ✅ |
| 项目环境 / 模板 / 通知 | 协助 | 协助 | ✅ |
| 用例 / 计划 / 缺陷日常操作 | ❌ | 一般不介入 | 协调，不替代测试 |

### 1.3 职责边界（Do / Don't）

| 角色 | ✅ 应该做 | ❌ 不应该做 |
|------|---------|-----------|
| **系统管理员** | 平台初始化、组织创建、企微同步、资源池、Token | 代替项目管理员分配项目内细粒度权限 |
| **组织管理员** | 组织成员、项目创建、组织模板、服务集成 | 修改系统级参数、删除其他组织数据 |
| **项目管理员** | 项目成员、环境、模板、版本、消息通知 | 创建组织、修改组织架构源数据 |

## 2. 权限模型与菜单入口

### 2.1 权限三级前缀

| 层级 | 权限前缀 | 示例 |
|------|---------|------|
| 系统级 | `SYSTEM_*` | `SYSTEM_ORGANIZATION_PROJECT:READ+ADD` |
| 组织级 | `ORGANIZATION_*` | `ORGANIZATION_MEMBER:READ+UPDATE` |
| 项目级 | `PROJECT_*` | `PROJECT_USER:READ+UPDATE` |

权限通过**用户组**绑定到用户，同一用户可在不同组织 / 项目拥有不同角色。

### 2.2 系统设置菜单（系统管理员）

| 功能 | 路径 | 核心权限 |
|------|------|---------|
| 系统用户 | `/setting/system/user` | `SYSTEM_USER:READ` |
| 用户组 | `/setting/system/usergroup` | `SYSTEM_USER_ROLE:READ` |
| 组织与项目 | `/setting/system/organization-and-project` | `SYSTEM_ORGANIZATION_PROJECT:READ` |
| 组织架构 | `/setting/system/org-structure` | `SYSTEM_ORGANIZATION_PROJECT:READ` |
| 系统参数 | `/setting/system/parameter` | `SYSTEM_PARAMETER_SETTING_*:READ` |
| 资源池 | `/setting/system/resourcePool` | `SYSTEM_TEST_RESOURCE_POOL:READ` |
| Agent 集成 | `/setting/system/agent-integration` | `SYSTEM_USER:READ` |
| 操作日志 | `/setting/system/log` | `SYSTEM_LOG:READ` |

### 2.3 组织设置菜单（组织管理员）

> 入口：进入目标组织后 → 左下角 / 头像 → **组织设置**

| 功能 | 路径 | 核心权限 |
|------|------|---------|
| 成员 | `/setting/organization/member` | `ORGANIZATION_MEMBER:READ` |
| 组织架构 | `/setting/organization/org-structure` | `ORGANIZATION_MEMBER:READ` |
| 用户组 | `/setting/organization/usergroup` | `ORGANIZATION_USER_ROLE:READ` |
| 组织项目 | `/setting/organization/project` | `ORGANIZATION_PROJECT:READ` |
| 服务集成 | `/setting/organization/serviceIntegration` | `SYSTEM_SERVICE_INTEGRATION:READ` |
| 组织模板 | `/setting/organization/template` | `ORGANIZATION_TEMPLATE:READ` |
| 组织日志 | `/setting/organization/log` | `ORGANIZATION_LOG:READ` |

### 2.4 项目设置菜单（项目管理员）

> 入口：进入目标项目后 → **项目设置**（`/project-management`）

| 功能 | 路径 | 说明 |
|------|------|------|
| 基本信息 | `/project-management/permission/basicInfo` | 项目名称、描述 |
| 菜单管理 | `/project-management/permission/menuManagement` | 项目内菜单可见性 |
| 项目版本 | `/project-management/permission/projectVersion` | 版本管理 |
| 成员 | `/project-management/permission/member` | 项目成员与角色 |
| 用户组 | `/project-management/permission/userGroup` | 项目级权限组 |
| 项目模板 | `/project-management/template` | 用例/缺陷/接口模板 |
| 环境管理 | `/project-management/environmental` | 测试环境 URL、变量 |
| 消息管理 | `/project-management/messageManagement` | 通知 Webhook |
| 文件管理 | `/project-management/fileManagement` | 项目文件 |
| 操作日志 | `/project-management/log` | 项目操作审计 |

## 3. 新环境 / 新组织接入 SOP

> **触发时机**：新环境首次部署 / 新业务线（新企微主体）接入  
> **主责**：系统管理员；组织管理员、项目管理员配合验收

### 3.1 环境就绪检查（系统管理员）

```
□ Step 1：确认服务可用
  ├─ 前端可访问（开发 http://localhost:5173，生产以实际域名为准）
  ├─ 后端可访问（默认 http://localhost:8081）
  └─ 管理员账号 admin 可登录

□ Step 2：确认自研版限制已解除
  ├─ 后端：CommunityLicenseService / CommunityUserXpackService 已生效
  ├─ 前端：VITE_MS_UNLIMITED=true
  └─ 验收：「创建组织」按钮可见，无「社区版仅支持 1 个组织」阻断

□ Step 3：确认中间件就绪（生产）
  ├─ MySQL / Redis / Kafka / MinIO
  └─ 资源池节点可连通（接口测试 / 性能测试需要）
```

### 3.2 首个 / 新增组织接入流程

```
Step 1：创建组织（系统管理员）
  路径：系统设置 → 组织与项目 → 创建组织
  填写：组织名称、组织管理员（至少 1 人）
  验收：列表出现新组织；模板 / 自定义字段 / Bug 状态流自动初始化

Step 2：配置组织架构数据源（系统管理员，可选其一）
  方案 A — 企微同步（推荐）：§5.2
  方案 B — Excel 导入：§5.3
  方案 C — 手动创建系统用户 + 组织成员：§9.1

Step 3：组织管理员进入组织
  操作：组织列表 →「进入组织」
  验收：左侧菜单切换为组织 / 项目上下文

Step 4：创建项目（组织管理员）
  路径：组织设置 → 组织项目 → 创建项目
  填写：项目名称、项目管理员
  验收：项目管理员可进入项目并看到用例 / 计划等菜单

Step 5：项目基础配置（项目管理员）
  ├─ 项目成员与角色（§8.1）
  ├─ 测试环境（§8.2）
  ├─ 模板确认（§8.3）
  └─ 消息通知（可选）

Step 6：业务团队验收
  ├─ 测试负责人确认可创建用例 / 计划
  ├─ PM 确认可进入项目并查看报告
  └─ 通知各角色可以开始迭代工作
```

### 3.3 接入验收 Checklist

- [ ] 多组织可创建、可切换
- [ ] 组织管理员可管理本组织成员与项目
- [ ] 项目管理员可分配成员并配置环境
- [ ] 普通成员登录后可见正确菜单
- [ ] 组织架构数据与 HR / 企微一致（如已配置同步）
- [ ] Agent 专用测试计划可由测试负责人创建（见测试人员 SOP）

## 4. 组织管理 SOP（系统管理员）

### 4.1 创建组织

**路径**：系统设置 → 组织与项目 → 创建组织

| 字段 | 规范 | 示例 |
|------|------|------|
| 组织名称 | 公司 / 事业部全称，唯一 | `米多科技` |
| 组织管理员 | 至少 1 人，赋 `org_admin` | 事业部 IT 负责人 |

**系统自动完成**（无需手工操作）：

- 组织级用例 / API / Bug **模板**初始化
- 组织级**自定义字段**初始化
- Bug **状态流**初始化
- 组织管理员绑定 `org_admin` 角色

**验收**：

- [ ] 组织列表可搜索到新组织
- [ ] 管理员可「进入组织」
- [ ] 组织设置 → 组织模板 有默认数据

### 4.2 编辑 / 启用 / 禁用组织

| 操作 | 路径 | 注意事项 |
|------|------|---------|
| 编辑名称 / 描述 | 组织与项目 → 编辑 | 名称变更不影响已有数据 ID |
| 禁用组织 | 组织与项目 → 禁用 | 成员无法进入该组织上下文 |
| 启用组织 | 同上 | 恢复访问 |
| 删除组织 | 组织与项目 → 删除 | **不可逆**；需确认无活跃项目或先归档 |

> **禁用 vs 删除**：人员短期离岗 / 业务暂停用**禁用**；组织永久下线才考虑**删除**（需数据备份与审批）。

### 4.3 组织切换

| 角色 | 切换入口 |
|------|---------|
| 系统管理员 | 系统设置 → 组织与项目 →「进入组织」；或头像 → 切换组织 |
| 组织管理员 | 头像 → 切换组织 |
| 普通用户 | 仅可切换到已有权限的组织 |

**切换后必须确认**：左上角组织名、项目列表、菜单权限与预期一致。

### 4.4 系统级「组织与项目」运维

| 场景 | 操作 |
|------|------|
| 查看全部组织 | 系统设置 → 组织与项目 |
| 代管某组织项目 | 进入该组织 → 组织设置 → 组织项目 |
| 跨组织问题排查 | 切换组织上下文后复现，禁止混用不同 orgId 的数据 |

## 5. 组织架构与成员同步 SOP

### 5.1 设计原则（管理人员必知）

| 原则 | 说明 |
|------|------|
| **企微为主数据源** | 部门 / 员工信息以企业微信为准，平台为**只读镜像** |
| **一企微主体 = 一 MS 组织** | 每个 MS 组织绑定独立企微 CorpID |
| **不在平台手工改部门** | 组织变更走企微 → 同步 → 平台更新 |
| **权责分离** | 组织架构管「是谁」；用户组 / 项目角色管「能做什么」 |

### 5.2 企微通讯录同步 SOP（系统管理员）

#### 5.2.1 前置准备

- [ ] 已在企微管理后台创建**通讯录同步**应用，获取 `CorpID`、`ContactSecret`
- [ ] 已创建对应 MS 组织（§4.1）
- [ ] 确认 MS 组织与企微主体一一对应

#### 5.2.2 配置步骤

```
Step 1：进入 系统设置 → 组织架构
Step 2：选择目标 MS 组织
Step 3：打开「同步配置」
  ├─ corpId：企微 CorpID
  ├─ contactSecret：通讯录 Secret
  ├─ scheduleEnabled：是否启用定时同步
  ├─ scheduleCron：如 0 0 2 * * ?（每天凌晨 2 点）
  └─ retryTimes：失败重试次数（默认 3）
Step 4：点击「测试连通性」→ 确认 deptCount 正常
Step 5：点击「手动同步」→ 首次全量同步
Step 6：查看同步日志，确认无大面积失败
```

#### 5.2.3 同步后验收

- [ ] 左侧部门树与企微架构一致
- [ ] 成员列表人数合理，无大面积「同步失败」
- [ ] 新入职员工在企微添加后，下次同步可在平台查到
- [ ] 离职员工在企微禁用后，同步状态 / 启用状态正确收敛

#### 5.2.4 日常运维

| 操作 | 触发时机 | 执行人 |
|------|---------|:--:|
| 手动同步 | 企微架构调整后、新人批量入职 | 系统管理员 |
| 查看同步日志 | 每次同步后 | 系统管理员 |
| 调整 Cron | 同步窗口冲突 / 频率不足 | 系统管理员 |
| 排查同步失败 | 日志有 ERROR | 系统管理员 + 企微管理员 |

**注意**：企微返回**空列表**时，同步引擎会**跳过失活**，防止误删。若出现「企微有数据但平台为空」，优先检查 Secret 权限与 CorpID 是否匹配。

### 5.3 Excel 导入组织架构 SOP（备选）

> **适用**：无企微、企微不可用、或初始化历史数据  
> **状态**：P3 能力，以实际环境功能开关为准

```
Step 1：系统设置 → 组织架构 → 下载导入模板
Step 2：按模板填写
  ├─ Sheet1 部门：组织名称、部门路径（/ 分隔）
  └─ Sheet2 用户：姓名、邮箱、组织名称、部门路径、用户组
Step 3：上传导入 → 查看成功 / 失败行报告
Step 4：失败行修正后重新导入
Step 5：组织管理员在「组织架构」页验收
```

**规则**：

- 组织名称必须已存在于 MS
- 部门路径同组织内唯一；先父后子
- 邮箱全局唯一

### 5.4 组织管理员查看组织架构

**路径**：组织设置 → 组织架构（`/setting/organization/org-structure`）

| 能力 | 说明 |
|------|------|
| 查看部门树 | 只读，含各部门人数 |
| 查看成员列表 | 按部门筛选、关键字搜索 |
| 查看成员详情 | 姓名、邮箱、部门、同步状态 |
| 触发同步 | **仅系统管理员**可操作（组织管理员一般为只读查看） |

## 6. 组织级治理 SOP（组织管理员）

### 6.1 组织成员管理

**路径**：组织设置 → 成员

#### 6.1.1 添加成员

```
Step 1：组织设置 → 成员 → 添加成员
Step 2：从系统用户列表选择（用户须已由系统管理员创建或同步产生）
Step 3：分配组织用户组（默认 org_member）
Step 4：保存 → 通知成员已开通组织权限
```

#### 6.1.2 成员角色调整

| 用户组 | 适用人员 | 典型权限 |
|--------|---------|---------|
| `org_admin` | 组织管理员 | 组织全部设置 |
| `org_member` | 普通组织成员 | 可被加入项目，无组织设置权限 |

#### 6.1.3 移除成员

```
确认成员已退出本组织全部项目（或已交接）
  → 组织设置 → 成员 → 移除
    → 确认该用户无法进入本组织上下文
```

> 移除组织成员**不会**删除系统用户账号；仅解除与本组织的关联。

### 6.2 组织用户组管理

**路径**：组织设置 → 用户组

| 操作 | 说明 |
|------|------|
| 查看内置组 | `org_admin`、`org_member` 等，谨慎修改权限 |
| 自定义用户组 | 按本组织需要新增（如「组织审计只读」） |
| 权限分配 | 勾选 `ORGANIZATION_*` 权限项 |

**建议**：非必要不修改内置 `org_admin` / `org_member` 权限集合，优先通过**项目角色**控制业务操作范围。

### 6.3 组织项目创建与归档

**路径**：组织设置 → 组织项目

#### 6.3.1 创建项目

| 字段 | 规范 | 示例 |
|------|------|------|
| 项目名称 | `{产品名}-{版本/线}` | `工单系统-V2` |
| 项目管理员 | 至少 1 人 | 测试负责人 |
| 描述 | 简要说明范围 | 米多工单系统 2026 主线 |

创建后系统自动初始化项目级模板与服务，项目管理员接手 §8 配置。

#### 6.3.2 项目生命周期

| 状态 | 操作 | 场景 |
|------|------|------|
| 进行中 | 正常维护 | 活跃迭代 |
| 暂停 | 禁用项目或限制新成员 | 业务暂停 |
| 归档 | 禁用 + 文档备份 | 产品下线 |
| 删除 | 组织管理员 / 系统管理员 | **需审批**，不可逆 |

### 6.4 组织模板管理

**路径**：组织设置 → 组织模板

| 模板类型 | 用途 | 管理人员动作 |
|---------|------|-------------|
| 用例模板 | 功能 / 接口用例默认字段 | 迭代前确认字段与 PM 对齐 |
| 缺陷模板 | 缺陷单字段 | 确认严重级别、复现环境字段 |
| 接口模板 | API 定义默认结构 | 与接口测试负责人确认 |
| 自定义字段 | 组织级扩展字段 | 新增字段需评估对历史数据影响 |
| 状态流 | Bug 状态流转 | 与研发流程对齐，变更需通知全员 |

**变更原则**：模板变更影响**本组织全部项目**，变更前需通知各项目管理员。

### 6.5 组织服务集成

**路径**：组织设置 → 服务集成

| 集成 | 用途 |
|------|------|
| 缺陷平台（Jira / Tapd 等） | 缺陷双向同步 |
| 项目管理 | 需求关联 |
| 其他插件 | 按插件文档配置 |

配置完成后，在**项目级**选择是否启用具体集成。

## 7. 项目管理 SOP（组织 / 项目管理员）

### 7.1 项目创建后的标准交接

组织管理员创建项目后，必须完成交接：

```
组织管理员创建项目并指定项目管理员
  → 项目管理员完成 §8 基础配置（3 个工作日内）
    → 拉通测试负责人确认成员名单
      → 通知 PM / 测试团队项目已就绪
```

**交接清单**（组织管理员 → 项目管理员）：

- [ ] 项目名称 / ID / 管理员账号
- [ ] 本迭代参与人员名单与角色
- [ ] 测试环境 URL 与账号（由运维或开发提供）
- [ ] 是否启用 Agent 协作（如是，测试负责人建 Agent 专用计划）
- [ ] 关联的需求 / 缺陷平台集成要求

### 7.2 项目成员与角色分配

**路径**：项目设置 → 成员（`/project-management/permission/member`）

#### 7.2.1 推荐角色配置

| 项目角色 | 建议人员 | 权限范围 |
|---------|---------|---------|
| 项目管理员 | 测试负责人 | 项目全部权限 |
| 测试成员 | 测试工程师 | 用例读写 + 计划执行 + 缺陷读写 |
| 开发成员 | 开发工程师 | 用例只读 + 缺陷读写 |
| 产品成员 | 产品经理 | 用例只读 + 评审 + 报告只读 |
| 只读成员 | 管理层 | 各模块 READ |

#### 7.2.2 添加成员流程

```
Step 1：确认用户已是本组织成员（否则先走 §6.1）
Step 2：项目设置 → 成员 → 添加成员
Step 3：选择用户 + 项目用户组
```

#### 7.2.3 项目用户组微调

**路径**：项目设置 → 用户组

当内置组不能满足需求时（如「仅缺陷读写」），可新建项目用户组并勾选 `PROJECT_*` 权限。**最小权限原则**：只授予完成工作所需的权限。

## 9. 成员入离职与权限变更 SOP

### 9.1 新成员接入（标准流程）

```
HR / 行政确认入职
  → 企微添加账号（推荐）
    → 系统管理员：企微同步 或 系统用户手动创建
      → 组织管理员：添加为组织成员
        → 项目管理员：添加为项目成员并分配角色
          → 成员登录验证：组织 / 项目 / 菜单正确
            → 测试负责人：安排用例 / 计划权限验收
```

**SLA 建议**：常规入职 **1 个工作日内**完成平台权限开通；紧急接入 **2 小时内**完成项目级权限。

### 9.2 成员离职 / 转岗

```
HR 通知离职 / 转岗
  → 项目管理员：从相关项目移除成员（或降级为只读至交接完成）
    → 组织管理员：从组织移除（如离开事业部）
      → 系统管理员：
          ├─ 禁用系统用户（如完全离职）
          ├─ 禁用其 Agent Token（如有）
          └─ 确认企微同步后状态一致
```

**安全检查**：

- [ ] 无 orphaned 项目管理员（唯一管理员需先交接）
- [ ] Agent Token 已禁用
- [ ] 系统用户已禁用或移除敏感用户组

### 9.3 权限变更

| 变更类型 | 执行人 | 操作 |
|---------|:--:|------|
| 升级为项目管理员 | 组织 / 项目管理员 | 项目设置 → 成员 → 调整用户组 |
| 跨项目调动 | 原项目管理员移除 + 新项目管理员添加 | 两个项目分别操作 |
| 升级为组织管理员 | 系统 / 组织管理员 | 组织设置 → 成员 → 赋 `org_admin` |
| 临时只读 | 项目管理员 | 换至只读用户组，迭代结束后恢复 |

## 10. Agent Token 与系统级扩展

> 详细 Agent 规范见：[SOP-MeterSphere-Agent协作平台操作规范.md](./SOP-MeterSphere-Agent协作平台操作规范.md)

### 10.1 Agent Token 管理（系统管理员）

**路径**：系统设置 → Agent 集成 → Token 管理

| 操作 | 说明 |
|------|------|
| 创建 Token | 填写名称、关联用户、默认项目、Scope、过期时间 |
| 复制明文 | **仅创建时一次**，通知申请人妥善保存 |
| 禁用 / 删除 | 人员离职或 Token 泄露时立即执行 |

**Scope 对照**：

| Scope | 能力 |
|-------|------|
| `FUNCTIONAL_READ` | 检索用例、模块列表 |
| `FUNCTIONAL_SUBMIT` | 回写执行结果 |
| `FUNCTIONAL_ALL` | 全部 Agent 操作 |

**管理人员审批要点**：

- 一人一 Token，名称含使用人 + 用途
- 最小权限：只读分析 vs 回写分开
- 默认项目与 `MS_TEST_PLAN_ID` 对应正确

**路径**：系统设置 → 资源池

| 自研版 | 不受社区版「单资源池」限制，按实际需要扩展 |
| 节点异常 | 检查节点连通性、并发配置 |

### 10.3 系统用户管理（系统管理员）

**路径**：系统设置 → 系统用户

| 操作 | 说明 |
|------|------|

| Excel 导入 | `/system/user/import` |
| 禁用用户 | 离职首选；保留审计记录 |
| 重置密码 | 按企业安全策略 |

## 11. 命名规范与治理原则

### 11.1 命名规范

| 对象 | 规范 | 示例 |
|------|------|------|
| 组织 | `{公司/事业部名}` | `米多科技` |
| 项目 | `{产品名}-{版本/线}` | `工单系统-V2` |
| 测试计划 | `{迭代}-{类型}-{日期}` | `Sprint12-功能测试-20260713` |
| Agent 专用计划 | `Agent-{类型}-{年份}` | `Agent-功能测试-2026` |
| 测试环境 | `{产品}-{环境}` | `工单系统-预发` |
| Agent Token | `{工具}-{姓名}-{用途}` | `Cursor-张三-功能测试` |

### 11.2 治理原则

| 原则 | 说明 |
|------|------|
| **最小权限** | 用户组只给完成工作所需的权限 |
| **组织隔离** | 不同事业部数据不交叉，禁止跨组织误操作 |
| **主数据单一** | 组织架构以企微为准，不在 MS 手工维护部门 |
| **变更可追溯** | 组织 / 项目级操作查操作日志 |
| **先交接后删除** | 禁用优先于删除；删除前必须备份与审批 |

### 11.3 定期治理（建议频率）

| 活动 | 频率 | 执行人 |
|------|------|:--:|
| 闲置项目审查 | 每季度 | 组织管理员 |
| 成员权限审计 | 每季度 | 组织 + 项目管理员 |
| Agent Token 轮换 | 每季度 | 系统管理员 |
| 企微同步日志检查 | 每月 | 系统管理员 |
| 资源池容量评估 | 每半年 | 系统管理员 |

## 12. 异常处理与升级机制

### 12.1 常见问题

| 现象 | 可能原因 | 处理 |
|------|---------|------|
| 看不到「创建组织」 | 社区版限制未解除 | 检查 `VITE_MS_UNLIMITED=true` 与后端 Xpack 实现 |
| 切换组织后项目为空 | 未进入正确组织 / 项目未创建 | 重新切换；组织管理员创建项目 |
| 成员无法进入项目 | 未加组织成员或未加项目成员 | 按 §9.1 补齐 |
| 组织架构为空 | 未同步 / Secret 错误 | 检查企微配置，手动同步 |
| 同步后人数不对 | 企微部门权限范围不足 | 调整通讯录应用可见范围 |
| 创建组织后模板缺失 | 初始化失败 | 查后端日志、`OrganizationInitService` |
| 用户超过 5 人报错 | Xpack 未生效 | 确认 `CommunityUserXpackServiceImpl` 已部署 |
| Agent Token 401 | Token 禁用或过期 | 系统管理员重新创建 |
| 操作后无日志 | 权限不足未执行成功 | 查浏览器 Network + 操作日志页 |

### 12.2 升级路径

```
管理人员发现问题
  → 判断影响范围（单人 / 单项目 / 单组织 / 全系统）
    → P0（全员无法工作）：立即通知系统管理员 + 研发负责人
    → P1（单组织 / 单项目阻塞）：组织管理员 4h 内响应
    → P2（可绕行）：记录缺陷，下一工作日处理
      → 未解决：提交 issue（附 orgId、projectId、操作步骤、截图）
```

## 13. 术语表

| 术语 | 说明 |
|------|------|
| **系统管理员** | 拥有 `SYSTEM_*` 权限，管理全局组织、用户、资源池 |
| **组织管理员** | 拥有 `org_admin`，管理本组织成员、项目、模板 |
| **项目管理员** | 拥有项目全部权限，管理成员、环境、模板 |
| **组织** | 业务隔离单元，通常对应一个企微主体 / 事业部 |
| **项目** | 测试工作单元，对应用例 / 计划 / 缺陷的容器 |
| **用户组** | 权限集合，分系统 / 组织 / 项目三级 |
| **组织架构** | 企微通讯录的本地只读镜像（部门树 + 成员） |
| **资源池** | 接口 / 性能测试的执行节点资源 |
| **Agent Token** | 外部 AI Agent 接入凭据，格式 `msat_xxx` |
| **Xpack / License 解除** | 自研版解除社区版组织数、用户数、资源池限制 |
